# Telegram юзербот-парсер

Проект написан по ТЗ юзербот.docx: Pyrogram-юзербот собирает активных пользователей из истории Telegram-групп, а Aiogram-бот предоставляет админ-панель для управления парсингом, донорами, глубиной сбора и выгрузками.

## Запуск

1. Установите зависимости:

    venv\Scripts\python.exe -m pip install -r requirements.txt

2. Создайте .env на основе .env.example и заполните минимум:

    BOT_TOKEN=...
    ADMIN_ID=...
    API_ID=...
    API_HASH=...

ACCOUNT_DIR=Данные аккаунта Казахстан по умолчанию указывает на папку, где лежат .session и .json. Если SESSION_NAME не указан, используется первый .session файл из этой папки.

3. Если нужен прокси, укажите его в Прокси.txt или заполните PROXY в .env.

4. Запустите бота:

    venv\Scripts\python.exe main.py

## Админ-панель

Админ-панель доступна только пользователю с Telegram ID из .env (ADMIN_ID).

Функции:

- глубина парсинга: 1, 3, 7, 14, 30 дней или ручной ввод;
- добавление донора: @username, t.me/..., t.me/+..., joinchat/...;
- удаление донора;
- импорт групп, в которых состоит аккаунт;
- ручной запуск парсинга;
- ежедневный автоматический парсинг по времени AUTO_PARSE_TIME из .env;
- выгрузка только пользователей со статусом New с переводом в Exported;
- выгрузка всей базы.

## Данные

SQLite база по умолчанию: data/userbot_parser.sqlite3.

Выгрузки по умолчанию сохраняются в exports/.
